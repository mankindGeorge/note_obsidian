# 操作系统测验

## Question 1
在操作系统的目标中，“有效性”通常指的是什么？

- [x] 提高系统资源的利用率和系统的吞吐量
- [ ] 使计算机系统更容易被普通用户使用
- [ ] 遵循统一的国际标准，便于软件移植
- [ ] 能够方便地增加新的功能模块

**Hint:** 考虑操作系统在管理昂贵硬件资源时的核心诉求。

## Question 2
脱机输入/输出（Off-line I/O）技术主要解决了早期计算机系统中的哪个瓶颈？

- [x] CPU 等待人工操作导致的低效
- [ ] 多个作业在内存中竞争资源的问题
- [ ] 无法实现人机交互的问题
- [ ] 作业由于资源不足而无法运行的问题

**Hint:** 关注主机 CPU 与外部设备速度不匹配的问题。

## Question 3
多道批处理系统相比单道批处理系统，最显著的优点是：

- [x] 资源利用率显著提高且系统吞吐量大
- [ ] 作业周转时间大大缩短
- [ ] 用户可以随时干预程序的运行过程
- [ ] 简化了操作系统的设计难度

**Hint:** 思考当一个作业在进行 I/O 操作时，其他作业在做什么。

## Question 4
分时系统的核心特征中，“及时性”通常是指：

- [x] 用户的请求能在较短时间内得到响应
- [ ] 系统必须在严格的截止时间内完成任务
- [ ] 多个用户感觉自己在独占主机资源
- [ ] 系统能同时处理来自多路终端的数据

**Hint:** 考虑用户站在终端前等待系统回复时的心理预期。

## Question 5
在实时系统中，被称为“硬实时任务”的要求是：

- [x] 必须在规定的截止时间内完成，否则后果严重
- [ ] 允许偶尔错过截止时间，只要平均性能达标
- [ ] 任务必须具有固定的周期性执行规律
- [ ] 要求系统具有极强的交互控制能力

**Hint:** 思考一旦超过预定时间，系统是否还能维持正常功能。

## Question 6
并发性（Concurrence）和并行性（Parallelism）在操作系统中的主要区别在于：

- [x] 并发是在同一时间间隔内发生，并行是在同一时刻发生
- [ ] 并发需要多核处理器，并行可以在单核上实现
- [ ] 并发指物理上的同时，并行指逻辑上的同时
- [ ] 并发处理的是作业，并行处理的是进程

**Hint:** 注意“同一时刻”与“同一时间段”的细微差别。

## Question 7
操作系统的“共享”特征中，打印机这种设备通常属于哪种共享方式？

- [x] 互斥共享方式
- [ ] 同时访问方式
- [ ] 时分复用方式
- [ ] 虚拟分配方式

**Hint:** 想一想如果两个程序同时向同一台打印机发送数据，结果会怎样。

## Question 8
进程在执行过程中表现出的“异步性”（Asynchronism）是因为：

- [x] 由于资源限制和调度策略，进程的执行速度不可预知
- [ ] 多个进程在同一时刻并行处理不同的数据
- [ ] 程序指令本身的逻辑顺序存在分支
- [ ] 硬件故障导致程序意外中断

**Hint:** 考虑在资源有限的竞争环境下，进程是否能一气呵成地运行完毕。

## Question 9
在存储器管理中，将“逻辑地址”转换为“物理地址”的过程称为：

- [x] 地址映射
- [ ] 内存分配
- [ ] 虚拟存储
- [ ] 内存保护

**Hint:** 这个术语涉及到“映射”或“重定位”的概念。

## Question 10
微内核操作系统的结构设计采用了“机制与策略分离”的原理，关于这一原理描述正确的是：

- [x] 机制放在内核中，而策略实现在内核外的服务器中
- [ ] 策略放在内核中以保证执行速度
- [ ] 内核只负责执行，不提供任何机制
- [ ] 机制和策略都应该是面向对象的逻辑实体

**Hint:** 内核应该尽量保持精简，只保留最基础、最通用的功能。
