# 操作系统测验

## Question 1
在操作系统的目标中，哪一个目标旨在提高系统资源利用率和系统吞吐量？

- [x] 有效性
- [ ] 方便性
- [ ] 可扩充性
- [ ] 开放性

**Hint:** 考虑“提高产出”和“减少闲置”对应的管理目标。

## Question 2

- [x] 减少 CPU 的空闲时间
- [ ] 实现真正的多任务并行
- [ ] 支持多个用户交互
- [ ] 完全取消人工操作

**Hint:** 回想早期系统如何解决“高速主机”与“低速外设”之间的矛盾。

## Question 3
与单道批处理系统相比，多道批处理系统的主要优点是：

- [x] 资源利用率高且系统吞吐量大

- [ ] 大幅缩短了作业的平均周转时间
- [ ] 简化了内存管理和保护

**Hint:** 考虑多道程序环境对 CPU 繁忙程度的影响。

## Question 4

- [x] 采用时间片轮转运行方式
- [ ] 引入高可靠性的硬实时机制
- [ ] 使用静态分配内存方式
- [ ] 仅允许单用户单任务运行

**Hint:** 思考如何让每个终端用户都感觉到自己“独占”主机。

## Question 5

- [x] 必须在规定的截止时间内完成任务
- [ ] 提供极强的人机对话界面
- [ ] 允许大量用户通过终端共享主机
- [ ] 仅处理周期性的实时任务

**Hint:** 该系统在“控制”和“响应”方面对时间有什么严苛要求？

## Question 6
在操作系统中，“并发（Concurrence）”与“并行（Parallelism）”的本质区别在于：

- [x] 并发是同一时间间隔发生，并行是同一时刻发生
- [ ] 并发需要多处理机支持，并行不需要
- [ ] 并发指物理实体，并行指逻辑对应物
- [ ] 并发是静态的，并行是动态的

**Hint:** 注意区分“一段时间内”与“某个瞬时”。

## Question 7
