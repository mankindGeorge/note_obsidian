# 项目规则

## Markdown 笔记格式规范

### 标题格式

1. **标题之间不需要空行**：连续的标题直接换行，不添加额外空行
   ```markdown
   ## 二级标题
   ### 三级标题
   正文内容...
   ```

2. **标题与正文之间空一行**：标题后面如果有正文内容，需要空一行
   ```markdown
   ### 标题
   正文内容
   ```

### 代码块格式

1. **代码块前后空一行**：代码块与上下文之间需要空行分隔
 

2. **代码块标记与语言名之间无空格**

### 标点与符号

1. **中文与英文/符号之间无空格**


### 章节分隔

1. **使用`---`分隔大章节**：不同主题的大章节之间使用分隔线
   ```markdown
   ## 章节 A
   内容...
   ---
   ## 章节 B
   内容...
   ```

### 列表格式

1. **列表项之间不空行**：同一列表的各项之间直接换行
   ```markdown
   - 项目 1
   - 项目 2
   - 项目 3
   ```

2. **嵌套列表缩进使用Tab**

### 强调格式

1. **参数名使用行内代码**：如`debug=True`
2. **重要内容使用加粗**：如**上线关闭**
3. **说明性文字使用加粗**：如**参数说明：**

### 文件命名

1. 笔记文件使用中文命名
2. 文件名与一级标题保持一致

## 代码规范

1. 代码中使用中文注释
2. 变量、函数、类等使用英文命名
3. 除非要求，不使用 try-except 语句

## 文件组织

1. 每种格式的文件保存在不同的文件夹中
2. Python 生成的图片统一放在专门放图片的文件夹里
