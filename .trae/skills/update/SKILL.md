# 更新 README.md 笔记导航 Skill

## 功能描述

自动扫描项目中的笔记文件，提取关键信息并更新 README.md 的笔记导航部分。

## 触发条件

- 用户要求"扫描笔记更新内容"
- 用户要求"更新 README.md"
- 用户要求"查看笔记内容，更新 README.md"

## 执行流程

### 1. 扫描项目目录结构

使用 `LS` 工具扫描项目根目录，识别所有笔记文件夹和文件。

**注意事项：**

- 忽略 `.obsidian` 目录（Obsidian 配置文件）
- 忽略 `.trae` 目录（Trae 配置文件）
- 忽略 `DayPlan` 目录（日常计划，非笔记）
- 忽略 `templates` 目录（模板文件，非笔记）

### 2. 检查笔记文件更新内容

使用 `Read` 工具读取每个笔记文件的内容，提取关键信息：

**提取信息：**

- 文件标题（一级标题）
- 主要章节（二级标题）
- 核心知识点（三级标题或重要内容）
- 文件是否为空（空文件不添加到 README.md）

**内容分析：**

- 识别笔记的主题和分类
- 提取关键词作为描述
- 确保描述简洁明了（不超过 20 字）

### 3. 更新 README.md

根据扫描结果更新 README.md 的笔记导航部分：

**更新规则：**

- 按分类组织笔记（DataCollection、DB、OS 等）
- 使用表格格式展示笔记链接和描述
- 描述应准确反映笔记内容
- 移除已删除的笔记链接
- 添加新增的笔记链接
- 更新已有笔记的描述（如果内容有变化）

**表格格式：**

```markdown
| 笔记 | 描述 |
|------|------|
| [笔记名称](相对路径) | 简洁描述 |
```

### 4. 验证更新结果

检查 README.md 更新是否正确：

- 确认所有笔记链接有效
- 确认描述准确反映笔记内容
- 确认格式符合项目规则

## 项目规则遵循

根据 `project_rules.md` 的要求：

### Markdown 格式

- 使用 `# / ## / ###` 标题
- 标题之间不空行
- 标题与正文空一行
- 代码块前后空一行
- 有序列表用于步骤
- 无序列表用于要点
- 列表项不空行
- Tab 缩进

### 文件规则

- 中文命名
- 文件名 = 一级标题
- 必须更新 README.md

## 错误处理

### 文件读取错误

- 如果文件无法读取，跳过该文件并记录错误
- 如果文件为空，不添加到 README.md

### 路径错误

- 使用相对路径（相对于项目根目录）
- 确保路径分隔符正确（Windows 使用 `/` 或 `\`）

### 格式错误

- 如果 README.md 格式不符合规则，先修复格式再更新
- 保持原有格式风格的一致性

## 性能优化

### 批量读取

- 使用多个 `Read` 工具并行读取多个文件
- 减少工具调用次数，提高效率

### 增量更新

- 只更新有变化的部分
- 避免完全重写 README.md

## 总结

这个 Skill 的核心价值：

1. 自动化笔记管理流程
2. 确保 README.md 始终反映最新的笔记内容
3. 提供清晰的笔记导航
4. 遵循项目规则和格式要求

通过这个 Skill，用户可以轻松维护笔记仓库的导航结构，无需手动更新 README.md。
