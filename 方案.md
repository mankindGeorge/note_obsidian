
# 图书个性化推荐系统——第一阶段：分布式大类粗筛前置评分与拓扑控流算法方案

## 1. 系统架构与交互流向
本方案采用“轻量化前端智能体（Coze平台）+ 独立自研后端服务器（算法与核心逻辑）+ 学校图书馆官网API（数据源）”的分布式架构设计。通过隔离表现层与计算层，解决数据资产安全、跨域请求限制、算法高并发性能消耗及万级馆藏数据检索超时的问题。

### 📌 完整链路拓扑流向
1. 表现层（Coze智能体入口）：通过多轮多Agent引导式对话，收集用户的专业、兴趣偏好及当前选书场景。
2. 逻辑层衔接：智能化前端将“专业”静态映射为“培养方案课程表”，并连同“兴趣画像”和“场景参数”组装为标准数据载荷（Payload），通过网络接口（HTTP POST）发送至自研后端服务器。
3. 计算层（自研服务器）：触发前置大类评分算法，计算各中图号（CLC）得分，完成配额截断。
4. 数据层检索（自研服务器）：由后端服务器异步并发呼叫学校图书馆官网的公开搜索页API，获取具体书籍元数据。

## 2. 输入特征与网络接口数据契约（Payload）
当前端智能体完成用户画像引导后，向后端服务器发送的 JSON 数据报文结构如下：

```json
{
  "user_id": "string",
  "academic_courses": ["数据结构", "概率论"],
  "interest_raw_text": "对量化交易和全栈Web开发感兴趣，喜欢实战源码类的硬核书",
  "scenario": "semester_rush", 
  "weight_settings": {
    "w_academic": 0.6,
    "w_interest": 0.4
  }
}
````

- 特征释义：
    
    - `academic_courses`：智能体根据专业自动解耦出的当前学期培养方案核心课程。
    - `scenario`：离散型场景枚举值（如：semester_rush 代表期末突击，project_drive 代表课外项目驱动）。
    - `weight_settings`：用户显式调节或智能体默认分配的精力分配权重，满足：$$W_{academic} + W_{interest} = 1.0$$

## 3. 服务器端核心算法流水线

### 步骤一：LLM 语义路由与兴趣中图号泛化发掘

自研服务器在接收到报文后，首先调用轻量化大模型接口（或本地大模型），将模糊多维的 `interest_raw_text` 翻译为精确的中图分类号，并赋予原始兴趣关联度系数 $Base_{interest}$（取值范围为 $[0.0, 1.0]$）。

- 服务器内部LLM解析产出示例：
    
    - 量化交易 $\rightarrow$ `F830`（金融投资），$Base_{interest} = 1.0$
    - 全栈Web开发 $\rightarrow$ `TP393.09`（计算机网络程序设计），$Base_{interest} = 0.9$

### 步骤二：多变量融合计算模型（前置大类评分公式）

后端服务器将培养方案学业池（映射为基础学业系数 $Base_{academic} = 1.0$）与LLM生成的兴趣池进行合并。针对每一个涉案的中图号大类 $clc$，运行情境震荡评分公式，计算其最终的类目分值 $Category\_Score$：

$$Category\_Score(clc) = (Base_{academic}(clc) \times W_{academic} \times S_{academic\_shock}) + (Base_{interest}(clc) \times W_{interest} \times S_{interest\_shock}) + Bonus_{cross}$$

变量运算边界逻辑：

- $Base_{academic}(clc)$：若该中图号存在于当前学期学业刚性池中，取值为 $1.0$；否则为 $0$。
    
- $Base_{interest}(clc)$：由步骤一中LLM解析出的关联度得分（$0.0 \sim 1.0$）；若该类目与兴趣无关，取值为 $0$。
    
- $S_{academic\_shock} / S_{interest\_shock}$（情境震荡因子）：根据当前选书场景动态改变的杠杆系数，用于实现场景的硬性对抗与压制。
    
- $Bonus_{cross}$（交叉学科共鸣分）：固定偏置增益。若一个中图号同时存在于学业池和兴趣池中（即 $Base_{academic} > 0$ 且 $Base_{interest} > 0$），则 $Bonus_{cross} = 0.5$；否则为 $0$。
    

### 步骤三：情境震荡因子的级联配置矩阵

为了保证在极端时间节点（如期末）推荐结果不脱靶，服务器内置以下震荡矩阵：

1. 期末突击场景（`semester_rush`）：
    
    设置 $S_{academic\_shock} = 2.5$，$S_{interest\_shock} = 0.1$。
    
    效果：极端压制兴趣类中图号的分数，使配额全额向专业课教材及教辅类目倾斜。
    
2. 课外搞项目/打比赛场景（`project_drive`）：
    
    设置 $S_{academic\_shock} = 1.0$，$S_{interest\_shock} = 2.0$。
    
    效果：放大技术应用类目权重，鼓励跨学科算法和前沿应用技术的打捞。
    
3. 平时闲暇充能场景（`leisure_learning`）：
    
    设置 $S_{academic\_shock} = 1.0$，$S_{interest\_shock} = 1.0$。
    
    效果：严格按照用户设定的默认权重比例进行中图号分配。
    

### 步骤四：配额动态分配与双轨制图馆 API 爬取

服务器计算出所有涉及中图号的最终得分后，在内存中执行精排截断与动态异步网络请求的生成：

1. 降序排列与截断：将所有中图号按 $Category\_Score$ 从大到小排序，保留前 Top 3 的中图号。
    
2. 配额比例切分：设定单次请求的总抓取图书上限为 $N$ 本（通常设定 $N = 60$）。根据 Top 3 中图号的分数比例，动态计算各自的抓取件数 $Crawl\_Quota$。
    
3. 融入 1/3 随机探索的双轨抓取策略：
    
    对于每一个分配到 $Crawl\_Quota$ 的中图号，服务器向学校图书馆公开 API 发起双轨并行请求：
    
    - 确定性轨迹（占 2/3 配额）：服务器请求图书馆 API 的 `page=1` 及 `sort=borrow_count`（或综合默认排序），直接抓取该类目最靠前的核心经典书籍。
        
    - 随机探索性轨迹（占 1/3 配额）：服务器首先通过首次请求的返回体获取该中图号下的总页数 $Total\_Page$，在服务器内存中生成一个随机页码：
        
        $$random\_page = \text{random.randint}(3, \min(50, Total\_Page))$$
        
        直接请求该随机页，盲抽该大类下的冷门长尾书或刚上架新书。
        

## 4. 架构工程优势复盘

1. 规避前端跨域与性能灾难：图书馆官网 API 通常具备严格的跨域限制（CORS）或频率限制。由自研服务器充当代理和爬虫宿主，不仅能利用服务器级带宽加速网络请求，更能有效隐藏爬虫指纹。同时，万级数据在调用 API 前已被精准压缩至 60 本，网络吞吐量极小。
    
2. 平台解耦与逻辑自主：Coze等智能体平台仅承担前端 UI 交互与用户引导的“看板”角色。核心的图谱映射、数学公式算分、爬虫调度逻辑全部沉淀在自研服务器上。系统不依赖特定 AI 平台的内置插件，具备极强的架构迁移能力与商业扩展性。
    
3. 探索性机制的无缝衔接：步骤四中盲抽的 1/3 探索书被严格锁死在正确的中图号大类中。这些书在服务器内存中进入第二阶段（多变量二筛）时，如果不匹配用户的能力等级（如大牛拿到小白书）或不符合具体场景描述，会被二筛规则自动清除，绝不影响最终发送回智能体前端的 Top 10 输出质量。
