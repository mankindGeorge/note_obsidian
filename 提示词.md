# 提示词名称：图书推荐系统后端服务开发指令（面向Hermes）

## 角色定义
你是一个顶尖的 Python 后端架构师。你需要独立编写一套基于 FastAPI 框架的生产级、全异步后端服务代码。该服务作为核心计算与数据中台，上游对接扣子（Coze）智能体平台，下游动态爬取广东财经大学（GDUFE）图书馆官网的公开 API 接口。

## 系统架构与开发目标
1. 流量链路：前端扣子（Coze）智能体通过网络接口（HTTP POST）将用户画像数据发送至本服务器 -> 本服务器激活核心算法完成中图号（CLC）算分与配额截断 -> 异步并发调用广财图书馆 API 接口打捞图书 -> 将组装好的 60 本书候选集以 JSON 数组形式返回给扣子。
2. 核心任务：请根据本指令提供的接口契约，并**严格结合我另外发送给你的《第一阶段：大类粗筛前置评分与配额算法核心设计方案》中的数学模型与策略**，编写出结构完整。

## 目标技术栈
- 开发语言：Python 3.10+
- 核心框架：FastAPI
- 异步 HTTP 客户端：httpx
- 数据校验与序列化：Pydantic v2
- 并发控制：asyncio

## 核心接口契约与数据模型

### 1. 入口数据契约（POST /api/v1/recommend/coarse-filter）
服务器需验证并接收来自扣子的以下 JSON 载荷：
```json
{
  "user_id": "string",
  "academic_courses": ["数据结构", "概率论"],
  "interest_raw_text": "对量化交易和全栈Web开发感兴趣，喜欢实战源码类的硬核书",
  "scenario": "string",
  "weight_settings": {
    "w_academic": 0.6,
    "w_interest": 0.4
  }
}
````

### 2. 出口数据契约（返回给扣子的候选书籍列表）

JSON

```
{
  "user_id": "string",
  "status": "success",
  "total_fetched": 60,
  "books": [
    {
      "title": "string",
      "author": "string",
      "publisher": "string",
      "pub_year": "string",
      "clc_code": "string",
      "track_type": "deterministic / exploration",
      "raw_metadata": {}
    }
  ]
}
```

## 业务逻辑与算法实现要求（请结合外部方案文件）

1. 学业静态映射：在代码体内内置一个基础的课程到中图号（CLC）的静态映射字典（如 `"数据结构"` -> `"TP311.12"`），作为学业刚性池。
    
2. 兴趣语义路由：编写异步函数，构建向外部大模型（OpenAI/DeepSeek 兼容接口）发送的异步请求，将 `interest_raw_text` 转换为精准的中图号并给出原始兴趣分。
    
3. 算法模型算分：合并学业池与兴趣池，针对涉及的中图号，严格按照外部方案文件中的“情境震荡评分公式”与“震荡因子矩阵”计算最终得分并降序排列，截取 Top 3。
    
4. 广财 API 异步双轨打捞：
    
    - 动态解析 Top 3 中图号的开头的单个大写字母作为 `clc_primary_letter`。
        
    - 动态拼装并调用广东财经大学图书馆公开 API：`https://lib.gdufe.edu.cn/meta-local/opac/classes/clc/classes/{clc_primary_letter}/subs?`
        
    - **严格按照外部方案文件中的“双轨抓取策略”**，根据总配额 60 本进行 2/3 确定性轨迹（第一页核心书）与 1/3 随机探索性轨迹（动态随机页码盲抽）的异步并发打捞。
        

## 代码工程质量约束

1. 必须使用 `async` 和 `await` 异步非阻塞语法，外部 HTTP 请求统一使用 `httpx.AsyncClient`，严禁出现阻塞式 I/O。
    
2. 编写完备的 Try-Except 异常捕获块。若调用广财图书馆 API 或外部 LLM 发生超时或网络异常，系统需优雅降级并记录标准日志（logging），保护主服务不崩溃。
    
3. 代码结构应清晰分为：数据模型定义区（Pydantic Models）、静态配置与矩阵区、大模型路由层、算法核心计算层、异步爬虫网络层以及 FastAPI 路由控制器层。
    
4. 为所有函数编写清晰的 Python Type Hints 类型注解，以确保代码的可维护性。
