# 存储过程

## 概念

存储过程是存储在数据库服务器中的一段预编译的SQL语句集合，是一个可重用的数据库对象。

## 优点

- 模块化设计：一次创建，多次调用
- 提高性能：预编译，无需重复解析
- 减少网络流量：客户端只需发送调用指令
- 提高安全性：可设置权限，限制用户操作

## 创建存储过程

### 基本语法

```sql
CREATE PROCEDURE 过程名([参数列表])
BEGIN
    SQL语句序列
END
```

### 示例：插入物资数据

**题目：创建存储过程，向物资表中插入一条新记录。**

```sql
CREATE PROCEDURE InsertMaterial(
    IN mat_no VARCHAR(10),
    IN mat_name VARCHAR(50),
    IN speci VARCHAR(30)
)
BEGIN
    INSERT INTO stock(mat_num, mat_name, speci)
    VALUES(mat_no, mat_name, speci);
END
```

### 示例：更新库存量

**题目：创建存储过程，根据物资编号更新库存数量。**

```sql
CREATE PROCEDURE UpdateStock(
    IN p_mat_num VARCHAR(10),
    IN p_amount DECIMAL(10,2)
)
BEGIN
    UPDATE stock
    SET amount = p_amount
    WHERE mat_num = p_mat_num;
END
```

### 示例：带条件判断的存储过程

**题目：创建存储过程，检查某物资的库存量是否低于10，低于则返回"库存不足"。**

```sql
CREATE PROCEDURE CheckStock(
    IN p_mat_num VARCHAR(10)
)
BEGIN
    DECLARE msg VARCHAR(100);

    IF EXISTS(
        SELECT * FROM stock
        WHERE mat_num = p_mat_num AND amount < 10
    ) THEN
        SELECT '库存不足' AS message;
    ELSE
        SELECT '库存正常' AS message;
    END IF;
END
```

## 调用存储过程

```sql
CALL InsertMaterial('m005', '光纤跳线', 'SC-UPC/2.0');
CALL UpdateStock('m005', 100);
CALL CheckStock('m005');
```

## 删除存储过程

```sql
DROP PROCEDURE IF EXISTS InsertMaterial;
```

## 查看存储过程

```sql
SHOW PROCEDURE STATUS LIKE 'Insert%';
SHOW CREATE PROCEDURE InsertMaterial;
```

# 触发器

## 概念

触发器是一种特殊的存储过程，它在响应特定的数据库操作（INSERT、UPDATE、DELETE）时自动执行。

## 触发时机

| 时机 | 说明 |
| --- | --- |
| BEFORE | 在操作执行之前触发 |
| AFTER | 在操作执行之后触发 |

## 创建触发器

### 基本语法

```sql
CREATE TRIGGER 触发器名
[BEFORE | AFTER] [INSERT | UPDATE | DELETE]
ON 表名
FOR EACH ROW
BEGIN
    触发器语句
END
```

### 示例：自动记录变更日志

**题目：创建触发器，当向物资表插入新记录时，自动记录操作日志。**

```sql
CREATE TRIGGER MaterialLog
AFTER INSERT ON stock
FOR EACH ROW
BEGIN
    INSERT INTO stock_log(op_type, mat_num, op_time)
    VALUES('INSERT', NEW.mat_num, NOW());
END
```

### 示例：删除前备份

**题目：创建触发器，在删除物资记录前，将其备份到历史表中。**

```sql
CREATE TRIGGER MaterialBackup
BEFORE DELETE ON stock
FOR EACH ROW
BEGIN
    INSERT INTO stock_backup
    VALUES(OLD.mat_num, OLD.mat_name, OLD.amount, NOW());
END
```

### 示例：更新前检查

**题目：创建触发器，在更新库存数量时，自动将负数修正为0。**

```sql
CREATE TRIGGER CheckAmount
BEFORE UPDATE ON stock
FOR EACH ROW
BEGIN
    IF NEW.amount < 0 THEN
        SET NEW.amount = 0;
    END IF;
END
```

## NEW 与 OLD 对象

| 操作 | NEW 对象 | OLD 对象 |
| --- | --- | --- |
| INSERT | 插入的新数据 | 无 |
| UPDATE | 更新后的新数据 | 更新前的旧数据 |
| DELETE | 无 | 被删除的旧数据 |

## 查看触发器

```sql
SHOW TRIGGERS FROM 数据库名;
SHOW CREATE TRIGGER 触发器名;
```

## 删除触发器

```sql
DROP TRIGGER IF EXISTS 触发器名;
```

# 数据完整性

## 约束类型

| 约束 | 说明 | 关键字 |
| --- | --- | --- |
| 主键约束 | 唯一标识每行 | PRIMARY KEY |
| 外键约束 | 维护表间关系 | FOREIGN KEY |
| 唯一约束 | 字段值唯一 | UNIQUE |
| 非空约束 | 字段不能为空 | NOT NULL |
| 检查约束 | 字段值满足条件 | CHECK |
| 默认约束 | 字段默认值 | DEFAULT |
| 自动增长 | 数值自动递增 | AUTO_INCREMENT |

## 主键约束

### 创建时定义

**题目：创建物资表，物资编号为主键，名称不能为空。**

```sql
CREATE TABLE material (
    mat_num VARCHAR(10) PRIMARY KEY,
    mat_name VARCHAR(50) NOT NULL,
    speci VARCHAR(30)
);
```

### 复合主键

**题目：创建订单明细表，使用订单号和物资号联合作为主键。**

```sql
CREATE TABLE order_detail (
    order_id VARCHAR(10),
    mat_num VARCHAR(10),
    quantity INT,
    PRIMARY KEY(order_id, mat_num)
);
```

### 添加主键

```sql
ALTER TABLE material
ADD PRIMARY KEY(mat_num);
```

## 外键约束

### 作用

- 建立表间关联关系
- 实现参照完整性
- 防止非法数据进入

### 创建外键

**题目：创建领料表，物资编号和项目编号分别关联物资表和项目表。**

```sql
CREATE TABLE out_stock (
    out_id INT PRIMARY KEY AUTO_INCREMENT,
    mat_num VARCHAR(10),
    prj_num VARCHAR(10),
    quantity INT,
    FOREIGN KEY(mat_num) REFERENCES stock(mat_num),
    FOREIGN KEY(prj_num) REFERENCES salvaging(prj_num)
);
```

### 级联操作

| 选项 | 说明 |
| --- | --- |
| ON DELETE CASCADE | 删除主表记录时，同时删除从表相关记录 |
| ON DELETE SET NULL | 删除主表记录时，从表相关字段设为NULL |
| ON DELETE RESTRICT | 阻止删除（默认） |

### 级联示例

**题目：创建领料表，当删除物资或项目时，自动级联删除相关领料记录。**

```sql
CREATE TABLE out_stock (
    out_id INT PRIMARY KEY AUTO_INCREMENT,
    mat_num VARCHAR(10),
    prj_num VARCHAR(10),
    quantity INT,
    FOREIGN KEY(mat_num) REFERENCES stock(mat_num)
        ON DELETE CASCADE,
    FOREIGN KEY(prj_num) REFERENCES salvaging(prj_num)
        ON DELETE CASCADE
);
```

## 唯一约束

**题目：创建用户表，用户编号为主键，用户名和邮箱必须唯一。**

```sql
CREATE TABLE user (
    user_id VARCHAR(10) PRIMARY KEY,
    user_name VARCHAR(30) UNIQUE,
    email VARCHAR(50) UNIQUE
);
```

## 检查约束

### 基本语法

```sql
CHECK(条件表达式)
```

### 示例：限制数量范围

**题目：创建订单明细表，数量必须大于0且不超过1000，价格不能为负。**

```sql
CREATE TABLE order_detail (
    order_id VARCHAR(10),
    mat_num VARCHAR(10),
    quantity INT CHECK(quantity > 0 AND quantity <= 1000),
    price DECIMAL(10,2) CHECK(price >= 0)
);
```

### 示例：限制仓库名称

**题目：创建库存表，仓库名称只能是指定的三选一。**

```sql
CREATE TABLE stock (
    warehouse VARCHAR(20) CHECK(warehouse IN ('供电局1#仓库', '供电局2#仓库', '罗湖仓库'))
);
```

## 默认值约束

**题目：创建订单明细表，订单日期默认为当前时间，状态默认为"待处理"。**

```sql
CREATE TABLE order_detail (
    order_id VARCHAR(10),
    order_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'pending'
);
```

# 事务控制

## 事务概念

事务是一组要么全部成功、要么全部失败的SQL语句。

## ACID 特性

| 特性 | 说明 |
| --- | --- |
| Atomic（原子性） | 事务是不可分割的整体 |
| Consistency（一致性） | 事务执行前后数据库保持一致状态 |
| Isolation（隔离性） | 并发执行的事务互不干扰 |
| Durability（持久性） | 事务提交后，结果永久保存 |

## 控制语句

```sql
START TRANSACTION;  -- 开始事务
-- SQL语句
COMMIT;             -- 提交事务（永久保存）
ROLLBACK;           -- 回滚事务（撤销更改）
```

### 示例：安全更新库存

**题目：实现库存扣减操作，如果扣减后库存为负则回滚事务，否则提交。**

```sql
START TRANSACTION;

UPDATE stock SET amount = amount - 10 WHERE mat_num = 'm001';

IF (SELECT amount FROM stock WHERE mat_num = 'm001') < 0 THEN
    ROLLBACK;
ELSE
    COMMIT;
END IF;
```

# 章节小结

| 概念 | 作用 | 关键语句 |
| --- | --- | --- |
| **存储过程** | 预编译的SQL语句集合，模块化复用 | CREATE/ CALL/ DROP PROCEDURE |
| **触发器** | 响应特定操作的自动执行机制 | CREATE TRIGGER, NEW/OLD |
| **数据完整性** | 保证数据的准确性和一致性 | PRIMARY/FOREIGN/UNIQUE/CHECK |
| **事务控制** | 确保一组操作原子性执行 | START/COMMIT/ROLLBACK |
