# 结构化爬虫

是一种面向对象爬虫设计模式，将爬虫逻辑抽象为Content、Website、Crawler类，这种模式适用于同时抓取多个结构不同的网站，将结构统一格式化。

1. Content类（数据模型）
	- ​作用：定义抓取的目标字段，作为存储抓取结果的“容器”。它不关心数据从哪来，只关心数据长什么样。

```python
class Content:
    """所有页面共享的统一数据结构"""
    def __init__(self, url, title, body):
        self.url = url
        self.title = title
        self.body = body

    def print_data(self):
        print(f"URL: {self.url}")
        print(f"TITLE: {self.title}")
        print(f"BODY: {self.body}\n")
```

2. Website 类 (配置模型/Sitedata)
	- ​**作用**：存储特定网站的“元数据”。每个网站的 HTML 结构不同，我们将这些差异通过选择器记录在这里。

```python
class Website:
    """存储网站的结构信息"""
    def __init__(self, name, url, title_tag, body_tag):
        self.name = name
        self.url = url
        self.title_tag = title_tag  # 如 'h1' 或 'div.title'
        self.body_tag = body_tag    # 如 'div.post-content'
```

3. Crawler 类 (执行引擎)
	- ​**作用**：核心逻辑所在。它接收一个 Website 对象，利用 BS 库根据其定义的 title_tag 和 body_tag 进行抓取，并返回一个 Content 对象。

```python
import requests
from bs4 import BeautifulSoup

class Crawler:
    def __init__(self, site):
        self.site = site
        self.visited = []

    """通用爬虫引擎"""
    def get_page(self, url):
        try:
            req = requests.get(url)
            return BeautifulSoup(req.text, 'html.parser')
        except requests.exceptions.RequestException:
            return None

    def parse(self, site, url):
        """核心处理逻辑：将 Website 规则应用到具体 URL"""
        soup = self.get_page(url)
        if soup:
            # 使用 Website 类中定义的标签规则进行搜索
            title_node = soup.select_one(site.title_tag)
            body_node = soup.select_one(site.body_tag)
            
            title = title_node.text if title_node else "未找到标题"
            body = body_node.text if body_node else "未找到内容"
            
            return Content(url, title, body)
        return None
```

4. **总结：Sitedata 的管理建议**
	- ​将所有网站的配置集中管理，可以极大提高代码的可维护性：

```python
# 定义 Sitedata 列表
site_data = [
    Website("搜狐新闻", "https://sohu.com", "h1", "div.article-content"),
    Website("网易新闻", "https://163.com", "div.post_content_main h1", "div.post_body")
]

# 统一调度
crawler = Crawler()
for site in site_data:
    # 假设这里是搜索出的具体页面
    content = crawler.parse(site, "https://example.com/news/1.html")
    content.print_data()
```

- **优点**：
	- 逻辑（Crawler）与数据结构（Website/Sitedata）完全解耦。增加新网站时，只需添加一行 Website 配置，无需修改任何爬虫代码。

## 通过搜索抓取网站

这种模式模拟用户行为：在网站搜索框输入关键词，解析结果列表，并抓取每一条结果。

- **新增配置：** `Website` 类需增加 `search_url`（搜索接口）和 `result_listing`（结果链接的选择器）。
- **适用场景：** 只需要抓取特定主题（如“人工智能”）的相关数据，而非全站抓取。

```Python
class Website:
    def __init__(self, name, url, search_url, result_listing, result_url, title_tag, body_tag):
        self.name = name
        self.url = url
        self.search_url = search_url          # 搜索路径，如 "/search?q="
        self.result_listing = result_listing  # 结果列表的容器标签
        self.result_url = result_url          # 列表中指向详情页的标签
        self.title_tag = title_tag
        self.body_tag = body_tag

class Crawler:
    # ... 保留之前的 get_page 和 parse 方法 ...

    def search(self, topic, site):
        """在特定网站搜索并抓取所有结果"""
        soup = self.get_page(site.url + site.search_url + topic)
        results = soup.select(site.result_listing)
        
        for result in results:
            url = result.select_one(site.result_url).attrs['href']
            # 处理相对路径
            if site.url not in url:
                url = site.url + url
            content = self.parse(site, url)
            if content:
                content.print_data()
```

## 通过链接抓取网站

这种模式即传统的“爬虫脚本”，它从首页开始，识别所有符合正则表达式或特定规则的内链，递归地抓取整个网站。

- **新增配置：** `Website` 类需增加 `target_regex`（定义哪些链接是我们需要抓取的详情页）。
- **适用场景：** 监控新闻网站首页更新，或系统性地存档某个站点的所有文章。

```Python
import re

class Website:
    def __init__(self, name, url, target_regex, title_tag, body_tag):
        self.name = name
        self.url = url
        self.target_regex = target_regex # 正则表达式，如 r'^/articles/.*'
        self.title_tag = title_tag
        self.body_tag = body_tag

class Crawler:
    def __init__(self):
        self.visited = []

    def crawl(self, site):
        """发现并抓取网站内所有匹配的链接"""
        soup = self.get_page(site.url)
        # 寻找所有 a 标签
        links = soup.find_all('a', href=re.compile(site.target_regex))
        
        for link in links:
            link_url = link.attrs['href']
            if link_url not in self.visited:
                self.visited.append(link_url)
                # 拼接完整路径并抓取
                full_url = site.url + link_url if link_url.startswith('/') else link_url
                content = self.parse(site, full_url)
                if content:
                    content.print_data()
```

## 抓取多种类型的页面

现实中，一个网站（如知乎或京东）会有“文章页”、“讨论页”、“商品页”等。每种页面的 HTML 结构完全不同。

- **核心逻辑：** `Website` 不再是单一结构，而是一个 **“页面类型选择器”**。我们在抓取时先判断 URL 或页面特征，再选择对应的解析规则。

```Python
class Website:
    """复杂的网站配置，包含多种页面类型"""
    def __init__(self, name, url, title_tag):
        self.name = name
        self.url = url
        self.title_tag = title_tag 
        
class PageType(Website):
    """定义单一页面类型的解析规则"""
    def __init__(self, name, url, title_tag, page_info):
	    Website.__init__(self, name, url, title_tag)
        self.page_info = page_info

class Crawler:
    def parse_flexible(self, site, url):
        """根据页面内容自动选择解析器"""
        soup = self.get_page(url)
        
        # 尝试匹配每一个定义的页面类型
        for pt in site.page_types:
            title_node = soup.select_one(pt.title_tag)
            if title_node: # 如果能匹配到标题标签，说明找对了类型
                body_node = soup.select_one(pt.body_tag)
                title = title_node.text
                body = body_node.text if body_node else ""
                return Content(url, title, body)
        return None
```

> PageType作为Website的一个基类
## 笔记小结：如何选择结构？

| **结构类型**  | **核心驱动**        | **优势**                |
| --------- | --------------- | --------------------- |
| **搜索驱动**  | `search_url`    | 结果精准，节省流量，适合定向数据采集。   |
| **链接驱动**  | `regex` (正则表达式) | 自动化程度最高，适合全站数据备份。     |
| **多类型驱动** | `PageType`      | 鲁棒性强，能处理复杂的大型门户或电商网站。 |
