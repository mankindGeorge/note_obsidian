# 结构化爬虫

是一种面向对象爬虫设计模式，将爬虫逻辑抽象为Content、Website、Crawler类，这种模式适用于同时抓取多个结构不同的网站，将结构统一格式化。

1. Content类（数据模型）
- ​作用：定义抓取的目标字段，作为存储抓取结果的“容器”。它不关心数据从哪来，只关心数据长什么样。

```python
class Content:
    """所有页面共享的统一数据结构"""
    def __init__(self, url, title, body):
        self.url = url
        self.title = title
        self.body = body

    def print_data(self):
        print(f"URL: {self.url}")
        print(f"TITLE: {self.title}")
        print(f"BODY: {self.body}\n")
```

2. Website 类 (配置模型/Sitedata)
- ​**作用**：存储特定网站的“元数据”。每个网站的 HTML 结构不同，我们将这些差异通过选择器记录在这里。

```python
class Website:
    """存储网站的结构信息"""
    def __init__(self, name, url, title_tag, body_tag):
        self.name = name
        self.url = url
        self.title_tag = title_tag  # 如 'h1' 或 'div.title'
        self.body_tag = body_tag    # 如 'div.post-content'
```

3. Crawler 类 (执行引擎)

- ​**作用**：核心逻辑所在。它接收一个 Website 对象，利用 BS 库根据其定义的 title_tag 和 body_tag 进行抓取，并返回一个 Content 对象。
```python
import requests
from bs4 import BeautifulSoup

class Crawler:
    """通用爬虫引擎"""
    def get_page(self, url):
        try:
            req = requests.get(url)
            return BeautifulSoup(req.text, 'html.parser')
        except requests.exceptions.RequestException:
            return None

    def parse(self, site, url):
        """核心处理逻辑：将 Website 规则应用到具体 URL"""
        soup = self.get_page(url)
        if soup:
            # 使用 Website 类中定义的标签规则进行搜索
            title_node = soup.select_one(site.title_tag)
            body_node = soup.select_one(site.body_tag)
            
            title = title_node.text if title_node else "未找到标题"
            body = body_node.text if body_node else "未找到内容"
            
            return Content(url, title, body)
        return None
```

4. **总结：Sitedata 的管理建议**

- ​将所有网站的配置集中管理，可以极大提高代码的可维护性：

```python
# 定义 Sitedata 列表
site_data = [
    Website("搜狐新闻", "https://sohu.com", "h1", "div.article-content"),
    Website("网易新闻", "https://163.com", "div.post_content_main h1", "div.post_body")
]

# 统一调度
crawler = Crawler()
for site in site_data:
    # 假设这里是搜索出的具体页面
    content = crawler.parse(site, "https://example.com/news/1.html")
    content.print_data()
```

- **优点**：
	- 逻辑（Crawler）与数据结构（Website/Sitedata）完全解耦。增加新网站时，只需添加一行 Website 配置，无需修改任何爬虫代码。
